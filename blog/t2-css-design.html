Identity Crisis (Classes or IDs)

So you want to start using best practices regarding ID and Classes on CSS but you don’t know where to start, fear no more this is Identity crisis for dummies.

Lets start with the basics, the first things that you got to ask yourself when deciding is, Does this element that I’m given this attribute is unique or does it occur several times during the whole length of the HTML document? , You may have deduct the answers if it occurs several times give it a Class, if it happens only once during the whole document give it an ID. Simple as that we are done you don’t need to know anything else except that you do there are other factors that are necessary to consider, lets take a closer look.

So Classes allow you to quickly change the style to a lot of elements by grouping them in a logical way, in contrast ID lets you specifically target a particular element and change only its attributes. IS a good idea to give IDs to unique parts of your web page like the Header, footer, side bars and other containers in your site, because in this way you can target them specifically and also link them using anchors tags, functionality that is not possible to achieve with classes. On the other hand is helpful to give classes to reoccurring elements like articles, images, paragraph and headings so that you can easily modify them all and keep the integrity of the webpage.

IDs by definition have a higher rank than Classes witch means that and ID will always triumph a Class when a conflict occurs. This is helpful because for example we if we have defined a class that groups all elements that have a green background and a black border we can assign this same class to a new element and also and ID to it so now this new element has inherit the same black border as their peers in the class but can have a red background thanks to its ID.

So easy peasy when you are trying to decide weather to use an ID or a Class just follow the rule of thumb, is the element is unique and only happens once in the document go for an ID, if it is a recurring element go for a class, and don’t forget that id have a higher weight than classes and will override them.
